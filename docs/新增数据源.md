# 新增第三数据源

黑猫（`heimao`）与小红书（`xhs`）是 MVP 验证源。新增数据源时，只需实现 **CrawlAdapter + NormalizeAdapter** 并在注册表中登记，无需修改 IntelRecord schema、看板或 `/api/intel/*` 响应结构。

## 步骤

### 1. 配置 `config.json`

```json
"sources": {
  "weibo": {
    "enabled": false,
    "label": "微博"
  }
}
```

将 `enabled` 设为 `true` 后，该源会出现在监测任务来源列表与 `GET /api/sources`。

### 2. 实现 CrawlAdapter

在 `intel/sources/weibo.py`：

- 类属性 `source_id = 'weibo'`
- 方法 `crawl(self, crawl_ctx, task, partner, options) -> list[dict]`
- 返回原始 dict 列表（站点特有字段均可）
- 监测会话内使用 `managed_session=True` 调用爬虫，避免重复开关 Chrome
- 登录逻辑复用 `login_gate.py`，不要重复实现轮询

参考：`intel/sources/heimao.py`、`intel/sources/xhs.py`

### 3. 实现 NormalizeAdapter

在 `intel/normalizers/weibo.py`：

- 映射为 NormalizedRecord 最小字段：`source`, `external_id`, `url`, `title`, `body`, `published_at`, `author`, `extra`, `raw_payload`
- 源特有字段只放入 `extra`，不要扩展 IntelRecord 顶层字段

参考：`intel/normalizers/heimao.py`、`reports.structure_xhs_record`

### 4. 注册

在 `intel/registry.py` 的 `register_default_sources()` 中：

```python
registry.register_crawler('weibo', WeiboCrawlAdapter)
registry.register_normalizer('weibo', WeiboNormalizeAdapter)
```

### 5. 验证

1. `GET /api/sources` 包含新源
2. 创建 MonitorTask 勾选新源，手动 `POST /api/monitor/run`
3. 确认 `raw_records` 与 `intel_records` 中 `source` 字段正确

## 架构原则

| 层级 | 是否按源分叉 |
|------|----------------|
| CrawlAdapter | 是 |
| NormalizeAdapter | 是 |
| PartnerMatcher | 否 |
| AnalyzePipeline | 否 |
| 看板 / API / IntelRecord | 否 |

业务系统对 `source` 自行加权；本系统不算综合风险分。
