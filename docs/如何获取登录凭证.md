# 如何获取黑猫 / 小红书登录凭证

本爬虫使用 **Cookie** 作为登录凭证（不是账号密码）。Cookie 是浏览器登录后由网站下发的会话标识，注入到自动化 Chrome 后即可带登录态搜索与查看详情。

---

## 方式一：在爬虫 Chrome 中登录后导出（推荐）

1. 启动 Web 控制台，点击 **启动 Chrome**（或开始爬取时会自动启动）。
2. 在配置面板 **登录凭证** 中，点击对应站点的 **打开登录页**。
3. 在弹出的 Chrome 窗口中 **手动完成登录**（扫码 / 短信等）。
4. **等待页面自动跳回黑猫首页**（说明微博 SSO 完成），再点击 **导出 Cookie**。凭证会保存到：
   - 黑猫：`credentials/heimao_cookies.json`
   - 小红书：`credentials/xhs_cookies.json`
5. 点击 **保存配置**，下次爬取会自动注入这些 Cookie。

> 若已配置 `chrome.profile_dir` 且勾选「仅使用 Chrome 用户目录」，登录态会保存在用户目录中，可不导出 Cookie，但换机器或清目录后需重新登录。

---

## 方式二：从日常浏览器复制 Cookie

### 黑猫投诉（新浪 / 微博体系）

1. 用 Chrome 打开 [https://tousu.sina.com.cn/](https://tousu.sina.com.cn/) 并登录。
2. 按 `F12` → **Application（应用程序）** → **Cookies**。
3. 选择 `https://tousu.sina.com.cn` 或 `https://passport.weibo.cn` 等相关域名。
4. 将关键 Cookie 整理为 JSON 数组（Playwright 格式），例如：

```json
[
  {
    "name": "SUB",
    "value": "你的值",
    "domain": ".sina.com.cn",
    "path": "/",
    "secure": true,
    "httpOnly": true
  }
]
```

5. 粘贴到控制台 **登录凭证 → 黑猫 Cookie (JSON)**，保存。

常见与登录相关的名称（因站点更新可能变化）：`SUB`、`SUBP`、`SCF`、`SINAGLOBAL` 等。  
**重要：** 黑猫登录走微博 SSO，Cookie 在 `weibo.com` / `passport.weibo.com` 域名下。仅复制 `sina.com.cn` 的 Cookie **无法登录**。导出后控制台应显示「含微博 SUB」。

**注意：** 勿使用已失效的 `passport.weibo.cn/signin?entry=tousu`（会 404）。程序已改为黑猫官方使用的 `passport.weibo.com/sso/signin?entry=general&source=heimao`。

### 扫码后仍显示未登录？

1. 扫码后是否已**跳回黑猫首页**？未跳转完成时不要导出。  
2. 导出后点 **黑猫：登录诊断**，日志中 `浏览器SUB=true` 才算成功。  
3. 旧版本导出会丢掉微博 Cookie，请**重新导出**一次。  
4. 导出后**不要关闭**爬虫专用 Chrome；程序已改为导出后不断开浏览器。  
5. 若配置里曾粘贴过过期 Cookie，请清空「黑猫 Cookie JSON」后仅依赖导出文件，或勾选「仅使用 Chrome 用户目录」。

### 小红书

1. 用 Chrome 打开 [https://www.xiaohongshu.com/](https://www.xiaohongshu.com/) 并登录。
2. `F12` → **Application** → **Cookies** → `https://www.xiaohongshu.com`。
3. 导出为 JSON 数组，**至少**应包含（名称以实际为准）：
   - `web_session`
   - `webId`
   - `a1`（如有）
4. `domain` 填 `.xiaohongshu.com`，粘贴到 **小红书 Cookie (JSON)** 后保存。

也可使用扩展 **EditThisCookie**、**Cookie-Editor** 导出 JSON，再粘贴到本系统。

## 方式三：每行 name=value（需写 domain）

在配置文本框第一行写：

```
domain: .xiaohongshu.com
```

随后每行一个 Cookie：

```
web_session=xxxx
webId=yyyy
```

保存后会自动解析并写入 `credentials/*_cookies.json`。

## 爬取时自动等待登录（推荐）

勾选 **爬取详情页** 时，若检测到未登录，任务会 **暂停并等待** 你在已打开的 Chrome 中完成登录，成功后 **自动继续**，无需关闭浏览器。

### 黑猫投诉

1. 点击 **开始**（已勾选「爬取详情页」）。
2. 若未登录，页面顶部会出现 **等待黑猫投诉登录** 横幅，日志提示用 **微博扫码** 登录。
3. 在 Chrome 中完成扫码，**跳回黑猫首页** 后程序自动检测并续跑。
4. 若配置了「登录成功后自动导出 Cookie」，会写入 `credentials/heimao_cookies.json`。
5. 也可在配置里点 **黑猫：打开登录页** 提前登录。

### 小红书

1. 勾选 **尝试爬取详情页** 后开始爬取。
2. 未登录时同样会暂停，在 Chrome 中完成小红书登录（扫码等）后自动继续。
3. 详情页若仍为空，会再等待一次登录并重试该条详情。

### 仅抓列表、不登录

取消勾选 **爬取详情页** 时，默认 **不要求登录**，只抓搜索结果列表。若仍希望列表也必须在登录后抓取，可在 **登录凭证** 中勾选「未勾选详情时也要求先登录」。

### 超时

默认等待 **300 秒**（可在配置的「等待登录超时」修改）。超时后任务中止，请重新导出 Cookie 或延长超时后重试。

---

## 验证是否生效

- 控制台 **登录凭证** 区域会显示已加载 Cookie 条数（黑猫应显示「含微博 SUB」，小红书应显示「含 web_session」）。
- 开始爬取时日志应出现：`[heimao] 已注入 N 条 Cookie` 或 `登录成功` / `登录检测通过`。
- 若仍无搜索结果，请重新导出 Cookie（可能已过期），或勾选详情触发等待登录流程。

---

## 安全提示

- Cookie 等同于登录态，**不要**提交到公开仓库或发给他人。
- 建议将 `credentials/` 加入 `.gitignore`。
- Cookie 会过期，失效后按上述步骤重新导出即可。
